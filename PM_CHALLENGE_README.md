# PM Challenge Loop - Implementation Guide

## Overview

The **PM Challenge Loop** adds an adversarial Portfolio Manager agent that reviews equity research drafts before finalization. This ensures higher quality, more rigorous analysis by identifying blind spots, contradictions, and unsupported claims.

## Architecture

```
┌─────────────────────────────────────────────────────────────┐
│  PHASE 1: BASIC PM CHALLENGE (IMPLEMENTED)                  │
└─────────────────────────────────────────────────────────────┘

1. Specialist Agents (existing)
   ├─ analyze_filings
   ├─ analyze_financials
   ├─ analyze_news
   ├─ find_dcf_tool
   ├─ calculate_moving_average_tool
   └─ valuation_tool

2. Synthesis Agent (existing)
   └─ reporting_agent → generates draft report

3. PM Agent (NEW - Phase 1)
   ├─ Reviews draft
   ├─ Outputs structured critique (PMReview)
   └─ Generates Investment Committee Memo

┌─────────────────────────────────────────────────────────────┐
│  FUTURE PHASES (NOT YET IMPLEMENTED)                         │
└─────────────────────────────────────────────────────────────┘

Phase 2: Routing & Revision
  └─ PM → routes issues to specialists → updates draft → PM re-reviews

Phase 3: Hard Validation Rules
  └─ Numerical checks, contradiction detection

Phase 4: Learning & Improvement
  └─ Track patterns, improve prompts automatically
```

## Phase 1: What's Implemented

### ✅ Core Components

1. **Pydantic Schemas** (`pm_schemas.py`)
   - `PMReview` - Structured PM critique
   - `InvestmentCommitteeMemo` - Human-readable decision memo
   - `BlindSpot`, `RuleViolation`, `ContradictionFlag` types

2. **PM Agent** (`pm_agent.py`)
   - Reviews drafts with adversarial mindset
   - Scores 3 dimensions (1-5 scale):
     * Thesis Coherence
     * Numerical Precision
     * Qualitative-Quantitative Consistency
   - Identifies blind spots (7 taxonomies):
     * Assumption Risk
     * Narrative Drift
     * Evidence Mismatch
     * Qual/Quant Contradiction
     * Missing Scenario
     * Catalyst Ambiguity
     * Temporal Mismatch
   - Outputs verdict: APPROVE / REVISE / NEEDS_MORE_RESEARCH

3. **Orchestrator** (`reporting_pipeline.py`)
   - `generate_financial_report_with_pm()` function
   - Runs existing pipeline → PM review → saves outputs

4. **CLI Integration** (`main.py`)
   - `--enable-pm` flag to activate PM challenge

5. **Tracking** (`pm_improvements.txt`)
   - Log file for Phase 4 pattern analysis

## PM Review Dimensions

### 1. Thesis Coherence (Score: 1-5)

**What it checks:**
- Is there ONE clear investment stance?
- Do all sections support that stance?
- Internal contradictions?
- Recommendation matches valuation?

**Example issues:**
- "Conclusion says BUY but DCF shows 10% downside"
- "Bullish valuation but bearish sentiment with no reconciliation"

### 2. Numerical Precision (Score: 1-5)

**What it checks:**
- Revenue growth explicit?
- Margin assumptions stated?
- WACC components shown?
- Terminal growth justified?
- Scenarios (bear/base/bull) provided?
- Sensitivity analysis present?

**Example issues:**
- "Price target of $250 but no DCF output shown"
- "Margin expansion claimed but no operational bridge"
- "No sensitivity table for WACC or terminal growth"

### 3. Qualitative-Quantitative Consistency (Score: 1-5)

**What it checks:**
- News sentiment vs forecast alignment?
- Management commentary vs model assumptions?
- Macro risks reflected in valuation?
- Technical indicators vs fundamental thesis?

**Example issues:**
- "Bearish news but bullish DCF with no catalyst"
- "Management guided down but model assumes margin expansion"
- "Buy rating but stock -15% below 200-day MA with no catalyst"

## Blind Spot Taxonomy

### 1. Assumption Risk
**Model inputs unsupported or unrealistic**
- Growth exceeds guidance
- Margin expansion unjustified
- Terminal value heroic

### 2. Narrative Drift
**Conclusion tone stronger than evidence**
- "Strong Buy" but +5% upside
- High confidence despite wide ranges
- Risk underplayed

### 3. Evidence Mismatch
**Weak/stale/irrelevant evidence**
- Using 2-year-old data
- Cherry-picking one data point
- Missing key evidence

### 4. Qual/Quant Contradiction
**Qualitative signals conflict with model**
- Negative news + bullish DCF
- Management caution + model optimism
- Bearish technicals + bullish fundamentals

### 5. Missing Scenario
**Thesis fragile, no sensitivity**
- No bear case
- Binary risk unmodeled
- No sensitivity table

### 6. Catalyst Ambiguity
**Fair value but no path to realization**
- No catalyst identified
- Timeline vague
- Mean reversion assumed

### 7. Temporal Mismatch
**Long-term vs near-term inconsistency**
- Near-term headwinds ignored
- Long-term risks not modeled
- Cyclicality not addressed

---

## Verdict Logic

### APPROVE
- All dimension scores ≥3
- No CRITICAL rule violations
- Required outputs present
- Blind spots ≤2 MODERATE

### REVISE
- 1-2 dimension scores = 2
- ≤2 CRITICAL violations
- 1-2 required outputs missing
- Issues fixable with patches

### NEEDS_MORE_RESEARCH
- ≥1 dimension score = 1
- ≥3 CRITICAL violations
- ≥2 required outputs missing
- Major contradictions

## Troubleshooting

### Issue: PM review parsing fails

**Symptom:** `Warning: Failed to parse PM response`

**Fix:**
- PM agent returns JSON that we parse
- If LLM doesn't return valid JSON, fallback review is created
- Check raw response in console output
- May need to adjust PM system prompt for better JSON compliance

### Issue: PM is too strict

**Symptom:** Everything gets `NEEDS_MORE_RESEARCH`

**Fix:**
- Phase 1 PM may be overly adversarial
- Adjust scoring thresholds in `pm_agent.py`
- Tune system prompt to be more balanced

### Issue: PM is too lenient

**Symptom:** Everything gets `APPROVE`

**Fix:**
- Strengthen PM system prompt
- Add more hard rules (Phase 3)
- Check if draft quality is actually high

### Issue: IC memo is generic

**Symptom:** IC memo doesn't have specific details

**Fix:**
- PM needs more context from draft
- Improve IC memo generation prompt
- Pass specialist outputs to PM (Phase 2)

## FAQ

**Q: Why does PM review take so long?**
A: PM agent uses GPT-4o with 60s timeout. It's reviewing the entire draft carefully. Phase 2 will optimize this.

**Q: Can I use PM without generating a full report?**
A: Not in Phase 1. PM needs a complete draft to review. Future: could review partial drafts.

**Q: Does PM actually improve the report?**
A: Phase 1: No, PM only critiques. Phase 2 will add revision loop to actually fix issues.

**Q: How accurate is PM?**
A: PM is as good as the LLM (GPT-4o). It can miss subtle issues or be overly strict. Phase 4 will add learning to improve over time.

**Q: Can I customize PM's personality?**
A: Yes, edit `PM_SYSTEM_PROMPT` in `pm_agent.py`. You can make it more/less strict, change focus areas, etc.

## Example Workflow

### 1. Generate report with PM challenge

```bash
python main.py --company "Tesla" --ticker TSLA --year 2026 --enable-pm
```

### 2. Review PM outputs

```bash
# Quick view of verdict
grep "VERDICT:" pm_review.txt
grep "IC RECOMMENDATION:" ic_memo.txt

# Full PM review
less pm_review.txt

# Human-readable IC memo
less ic_memo.txt
```

### 3. Interpret results

**If verdict = APPROVE:**
- Report is ready for presentation
- IC memo lists strengths and invalidators
- Proceed to portfolio discussion

**If verdict = REVISE:**
- Review `BLIND SPOTS` section
- Check `AGENTS TO RE-QUERY`
- Phase 2: System will auto-fix these
- Phase 1: Manual fixes needed

**If verdict = NEEDS_MORE_RESEARCH:**
- Major issues found
- Review `RULE VIOLATIONS` (CRITICAL)
- Check `CONTRADICTIONS`
- May need to rethink thesis
